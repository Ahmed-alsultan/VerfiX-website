REST API Reference — v1

VerfiX REST API — Reference Documentation

The VerfiX API provides programmatic access to all VerfiX platform capabilities through a RESTful interface. All requests and responses use JSON. All endpoints require authentication via Bearer token.

Base URL and Environments

Environment Base URL Use
Production https://api.verfix.ch/v1 Live verification. Billed.
Sandbox https://sandbox.api.verfix.ch/v1 Integration testing. Not billed. Test data only.

Authentication

All API requests must include a Bearer token in the Authorization header:

Authorization: Bearer vfx_live_xxxxxxxxxxxxxxxxxxxxxxxx

API keys are issued per deployment and can be rotated from the VerfiX dashboard. Use vfx_test_ prefix for sandbox keys. API keys are environment-scoped — sandbox keys cannot be used in production.

Endpoints Overview

Method Endpoint Purpose
POST /v1/verify/document Verify an identity document (OCR, MRZ, security features)
POST /v1/verify/face Biometric face match with liveness detection
POST /v1/verify/voice Voice biometric verification for telephone authentication
POST /v1/verify/nfc NFC chip data extraction and validation from ePassport
POST /v1/kyb/company Business verification — company registry lookup and ownership mapping
POST /v1/kyb/ubo Identify and return ultimate beneficial owners above threshold
POST /v1/screen/aml AML, PEP, and sanctions screening for individual or entity
POST /v1/trust-score Generate composite Trust Score from provided verification IDs
GET /v1/verification/{id} Retrieve full verification record including audit trail
GET /v1/audit/{id} Retrieve audit trail for a specific verification event
POST /v1/webhooks Register a webhook endpoint for event notifications
GET /v1/webhooks List all registered webhook endpoints
DELETE /v1/webhooks/{id} Remove a webhook endpoint registration

Request and Response Format

POST /v1/verify/document — Example Request

Content-Type: application/json
{ "image_front": "<base64_encoded_image>", "image_back": "<base64_encoded_image>", "country": "CH", "document_type": "national_id" }

POST /v1/verify/document — Example Response

{ "verification_id": "vfy_3KvB9cH2mNpQrT", "status": "completed", "result": "approved", "document": { "type": "national_id", "country": "CH", "first_name": "Max", "last_name": "Mustermann", "date_of_birth": "1985-03-14", "document_number": "X1234567", "expiry_date": "2030-06-15", "mrz_valid": true }, "fraud_signals": { "is_tampered": false, "fraud_score": 3 }, "audit_id": "aud_7mK2pR9xQv", "processing_time_ms": 1247 }

Error Codes

HTTP Code Error Code Meaning Action
400 invalid_request Request body is malformed or missing required fields Check request body against API documentation
401 unauthorized Missing or invalid API key Verify Authorization header and key prefix
403 forbidden API key does not have permission for this endpoint Check key scopes in dashboard
422 unprocessable_image Image quality too low or format not supported Retry with higher quality image. Minimum 640x480px.
429 rate_limited Rate limit exceeded Retry after X-RateLimit-Reset header timestamp
500 internal_error Server error Retry with exponential backoff. Contact support if persistent.

Rate Limits

Default rate limits apply per API key. All responses include rate limit headers:

  • ›X-RateLimit-Limit: Maximum requests per window
  • ›X-RateLimit-Remaining: Requests remaining in current window
  • ›X-RateLimit-Reset: Unix timestamp when window resets

Default limit: 100 requests per minute per API key. Enterprise limits are negotiated as part of the service agreement. Contact sales for volume requirements.

Webhooks

VerfiX supports webhook notifications for asynchronous events. Register an endpoint via POST /v1/webhooks with a URL and event filter. VerfiX sends a POST request to the registered URL on each matching event.

Event Trigger
verification.completed Any verification (KYC, KYB, voice, NFC) has completed processing
verification.failed A verification could not be completed due to image quality or technical error
aml.hit An AML, PEP, or sanctions screening has returned a potential match
trust_score.generated A Trust Score has been generated for a set of verification results
monitoring.alert An ongoing monitoring rescan has detected a new match for an existing customer

Webhook requests include an X-VerfiX-Signature header containing an HMAC-SHA256 signature of the request body, signed with the webhook secret configured at registration. Verify this signature before processing webhook events.
