Security Controls

How we protect your data

Encryption

  • AES-256 encryption at rest
  • TLS 1.3 for all data in transit
  • Per-deployment key management
  • HSM-backed key storage

Data Residency

  • Swiss Tier III certified data centres
  • No cross-border transfers without consent
  • EU hosting option available
  • On-premise deployment supported

Access Control

  • Role-based access control (RBAC)
  • MFA enforced for all admin access
  • SSO support (SAML 2.0 / OIDC)
  • Session timeout and IP allowlisting

Audit Logs

  • Immutable event log per verification
  • Timestamped with cryptographic hash
  • FINMA-inspection ready export
  • 10-year retention available

Vulnerability Management

  • Annual third-party penetration testing
  • Continuous dependency scanning
  • Responsible disclosure policy
  • Report to security@verfix.ch

Incident Response

  • 72-hour breach notification
  • Documented IR runbooks
  • 24/7 security monitoring
  • Post-incident report to clients

FINMA Compliance Matrix

VerfiX addresses the following requirements of FINMA Circular 2016/7 on video and online identification.

Requirement Status How VerfiX meets it
Identity document verification ✓ Covered Document Verification API — 4,500+ document types, MRZ validation, security feature checks
Facial biometric match ✓ Covered Face Verification API — ISO 30107-3 liveness, biometric match score
Liveness detection ✓ Covered Active (blink/turn) and passive liveness detection, anti-spoofing
Audit trail per verification ✓ Covered Immutable timestamped log with cryptographic hash per event
Swiss data residency ✓ Covered All data processed and stored in Swiss Tier III certified data centres
Re-identification prevention ✓ Covered One-time session tokens, no persistent biometric storage by default
Staff training records ✓ Covered Available to customers on request via Trust Center documentation pack
