Infrastructure Security

Built on Swiss infrastructure — secure by design

Every VerfiX component is hosted in Switzerland. No personal data leaves Swiss territory. Our infrastructure is designed to meet the expectations of FINMA-supervised financial institutions.

Swiss Data ResidencyConfirmed

All processing and storage in Switzerland. AWS infrastructure in Zürich region (eu-central-2). No cross-border data transfers for personal data.

Encryption at RestAES-256

AES-256 encryption for all stored data. Field-level encryption for biometric processing data. Key management via AWS KMS with Swiss-region keys.

Encryption in TransitTLS 1.3

TLS 1.3 enforced on all API endpoints and web interfaces. HSTS with 2-year max-age. No support for TLS 1.0 or 1.1.

Access ControlsRBAC + MFA

Role-based access control (RBAC) for all internal systems. Multi-factor authentication mandatory for all VerfiX staff. Privileged access management (PAM) for production systems.

Penetration TestingAnnual

Annual third-party penetration tests by accredited security firms. Web application, API, and infrastructure scope. Findings remediated within defined SLA windows.

Security Monitoring

24/7 security event monitoring. SIEM with anomaly detection. Automated alerting for unusual API usage patterns, authentication anomalies, and data access events.

Data Protection

Privacy by design — nFADP and GDPR compliant

Biometric Data MinimisationnFADP Art. 6

Biometric templates and document images are deleted at session end by default. Only the non-biometric verification outcome is retained. Configurable retention for specific justified use cases.

Data Processing AgreementIncluded

Standard DPA provided to all customers before processing begins. Covers all GDPR Art. 28 and nFADP Art. 9 requirements. Enterprise customers may request customised DPA addenda.

Sub-Processor ManagementTransparent

Full sub-processor list maintained and published in the Trust Center. Customers notified 30 days before any sub-processor addition or material change.

Data Subject RightsGDPR Art. 15-20

API endpoints support access, rectification, erasure, and portability requests. VerfiX forwards direct data subject requests to the relevant customer within 5 business days.

Breach Notification72 hours

Confirmed breaches likely to result in risk to data subjects notified to customers within 72 hours. FDPIC notification guidance provided. Incident Response Policy published in Trust Center.

GDPR AdequacyEU Adequate

Switzerland holds EU GDPR adequacy status. No Standard Contractual Clauses required for EU organisations processing data in Swiss VerfiX infrastructure.

Responsible Disclosure

Security vulnerability disclosure policy

VerfiX operates a responsible disclosure programme. If you discover a security vulnerability in VerfiX systems, we ask that you report it to us before public disclosure, giving us reasonable time to investigate and remediate.

Report a Vulnerability
security@verfix.ch

We commit to acknowledging reports within 48 hours, providing a preliminary assessment within 7 days, and notifying you when the vulnerability has been remediated. We do not pursue legal action against researchers who follow this policy in good faith.
