Authentication

All API requests require a Bearer token in the Authorization header. You can generate API keys in the VerfiX Dashboard under Settings → API Keys.

Keep your API key secret. Never expose it in client-side code or public repositories.

API Key Format

HTTP Header
Authorization: Bearer vx_live_xxxxxxxxxxxxxxxxxxxx

Example Request

cURLCopy
curl -X POST https://api.verfix.ch/v1/kyc/verify -H "Authorization: Bearer vx_live_xxxx" -H "Content-Type: application/json" -d '{"document_type":"passport","country":"CH"}'

Test vs Live Keys

Key prefixEnvironmentBehavior
vx_test_TestNo real verifications, free, returns simulated results
vx_live_ProductionReal verifications, billed per usage

Errors

VerfiX uses standard HTTP status codes. All errors return a JSON body with code, message, and optional details.

StatusCodeMeaning
200okRequest succeeded
400invalid_requestMissing or invalid parameters
401unauthorizedInvalid or missing API key
402insufficient_creditsAccount has no remaining credits
429rate_limitedToo many requests — see rate limits
500server_errorInternal error — retry with backoff
Error Response
{ "error": { "code": "invalid_request", "message": "document_type is required", "request_id": "req_01HX7ZABCDEF" } }

Rate Limits

Rate limits apply per API key. Headers indicate current usage.

KYC — Verify Identity

Submit a document image and optional selfie to verify a person's identity. Returns a structured decision with confidence scores.

POST/kyc/verify

Request Parameters

ParameterTypeRequiredDescription
document_typestringrequiredpassport, id_card, driving_licence, residence_permit
document_frontbase64requiredBase64-encoded image of document front
document_backbase64optionalRequired for ID cards and driving licences
selfiebase64optionalSelfie for face match
countrystringoptionalISO 3166-1 alpha-2 country code (e.g. CH, DE, FR)
user_idstringoptionalYour internal user identifier for audit trail

Example Request

JSONCopy
{ "document_type": "passport", "document_front": "<base64>", "selfie": "<base64>", "country": "CH", "user_id": "usr_4829abc" }

Response

JSON Response
{ "verification_id": "ver_01HX7Z4QRSTUVWXY", "status": "approved", "trust_score": 96, "document": { "valid": true, "score": 95, "extracted": { "first_name": "Jane", "last_name": "Müller", "dob": "1985-04-12", "nationality": "CHE", "expires": "2031-08-22" } }, "face_match": { "score": 98, "liveness_passed": true }, "risk": { "level": "low", "flags": [] }, "created_at": "2025-06-11T10:23:45Z" }

KYB — Verify Business

Verify a business entity against commercial registries, check directors, UBO structure, and cross-reference against AML watchlists.

POST/kyb/verify

Request Parameters

ParameterTypeRequiredDescription
company_namestringrequiredRegistered legal name
registration_numberstringrequiredCompany registration / UID number
countrystringrequiredISO 3166-1 alpha-2 country code
include_ubobooleanoptionalInclude UBO structure analysis (default: false)
JSON
{ "company_name": "Muster AG", "registration_number": "CHE-123.456.789", "country": "CH", "include_ubo": true }

nationalitystringoptionalISO 3166-1 alpha-3 nationality code
Response

Webhook Events

VerfiX sends webhook notifications for key events. Configure webhook endpoints in the Dashboard under Settings → Webhooks.

Event Types

EventDescription
verification.completedA verification has completed with a final decision
verification.review_requiredVerification requires manual review
aml.hit_detectedAn AML screening returned a potential match
trust_score.updatedTrust Score recalculated (e.g., after AML refresh)

Webhook Payload

JSON
{ "id": "evt_01HX7ZXXXXXXXXX", "type": "verification.completed", "created": "2025-06-11T10:23:47Z", "data": { "verification_id": "ver_01HX7Z4QRSTUV", "status": "approved", "trust_score": 96 } }

Signature Verification

Webhooks are signed using HMAC-SHA256. Verify the X-VerfiX-Signature header on every request.

SDKs & Libraries

Official SDKs are in development. In the meantime, use the REST API directly with any HTTP client.

Node.js (REST)
// npm install node-fetch const res = await fetch('https://api.verfix.ch/v1/kyc/verify', { method: 'POST', headers: { 'Authorization': `Bearer ${process.env.VERFIX_API_KEY}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ document_type: 'passport', country: 'CH' }) }); const data = await res.json();
Python (REST)
# pip install requests import requests, os res = requests.post( 'https://api.verfix.ch/v1/kyc/verify', headers={'Authorization': f'Bearer {os.getenv("VERFIX_API_KEY")}'}, json={'document_type': 'passport', 'country': 'CH'} ) data = res.json()
