Identity Verification

Document Verification 101: How It Works and Why It Matters

How automated document verification works: the four layers from classification to tamper detection, MRZ validation, security feature checking, NFC chips, and why document library coverage determines digital KYC reach.

VerfiX AG 2026-06-19 Identity Verification

Document verification is the foundation of every KYC process. Before biometric matching, liveness detection, or AML screening, the identity document itself must be confirmed as genuine, unaltered, and belonging to a real individual. This guide explains how automated document verification works, what checks are performed, why document coverage breadth matters, and how NFC chip reading provides the highest available assurance. See also: Identity Verification FAQ, MRZ Verification, and NFC Chip Verification.

The four layers of document verification

1
Document Classification

The system identifies the document type (passport, national ID card, driving licence, residence permit), the issuing country, and the document version or generation. Each document type has a specific template defining the expected layout, security features, and data fields. Classification accuracy is the precondition for all subsequent verification steps.

2
MRZ Validation

The Machine Readable Zone — the two or three lines of structured data at the bottom of most identity documents — contains the holder's name, date of birth, document number, expiry date, nationality, and check digits. MRZ validation reads these fields, verifies the internal check digits (which are mathematically derived from the other fields), and confirms data consistency between the MRZ and the visual inspection zone.

3
Security Feature Verification

Modern identity documents contain multiple physical security features designed to prevent forgery: holograms, kinegrams, optically variable ink, microprint, UV-reactive patterns, laser perforations, and rainbow printing. Automated document verification checks for the presence and correct appearance of these features using UV, infrared, and visible-light capture, and cross-references against a database of expected features for each document type and version.

4
Tamper Detection

Beyond confirming security features, the system checks for signs of tampering: photo substitution (the photograph appears inconsistent with the surrounding document material), data field alteration (fonts or character spacing inconsistent with original printing), laminate deformation (physical signs of the protective layer having been lifted and replaced), and digital manipulation artefacts in captured images.

Document library coverage — why breadth matters

Switzerland has a resident population drawn from over 180 nationalities. A Swiss financial institution that can only verify Swiss passports and EU identity cards will be unable to serve a significant proportion of its potential customer base through its digital channel. Document library breadth — the number of document types, countries, and versions supported — directly determines what percentage of customers can be served digitally versus requiring in-branch identification. Global Document Coverage guide covers the specific challenges of non-EU document verification.

VerfiX document verification covers 4,500+ document types across 190+ countries, including all three generations of the Swiss Ausländerausweis (A, B, C, F, L, N, S permits), all current EU and EEA national identity documents, and the major Asian, African, and Americas document types. Coverage is continuously updated as new document versions are issued.

NFC chip reading — beyond visual verification

For documents containing NFC chips — ePassports, the German Personalausweis, the French CNIe, Italian CIE, and other modern national ID cards — NFC chip reading provides a verification layer that visual inspection cannot match. The chip contains the holder's data and biometric photograph signed by the issuing government's digital certificate. Validating the chip's cryptographic signature against the issuing country's certificate confirms that the chip data was genuinely issued by the government and has not been modified. Read the NFC chip verification guide.

Verification speed and accuracy targets

For FINMA Circular 2016/7 compliance and enterprise KYC at scale, document verification must complete in under three seconds of server-side processing time. Human-level accuracy for major document types should be at or above 99.5% for genuine documents. False rejection rate — the percentage of genuine documents incorrectly flagged — should be below 0.5% to avoid degrading conversion rates. VerfiX document verification meets these targets across all supported document types. See the Trust Center for the accuracy matrix by document category.

The weakest link in document verification is usually document library coverage rather than algorithmic accuracy. A system that achieves 99.9% accuracy on the 200 document types it supports is useless for the 4,300 types it does not. Institutions evaluating KYC vendors should specifically test coverage for their top-20 most common customer nationalities.
