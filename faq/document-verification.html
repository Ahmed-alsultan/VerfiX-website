Document Verification FAQ

Technical details of document verification: MRZ, NFC, security features, and fraud detection.

The Machine Readable Zone is the two or three line code at the bottom of passports and identity cards encoding personal data with check digits. VerfiX reads, validates check digits, and cross-references MRZ data against the visual inspection zone on every document. Check digit failure is a strong fraud indicator.

The Machine Readable Zone (MRZ) is a standardised block of text on identity documents following ICAO 9303 specifications: two lines of 44 characters on passports, three lines of 30 characters on identity cards. It encodes name, document number, nationality, date of birth, sex, expiry, and optional national data — each field followed by a mathematical check digit.

MRZ validation provides two security functions: (1) Data integrity: check digits allow mathematical verification that no character has been altered — even a single-character change in document number or date of birth will invalidate the check digit. (2) Data consistency: MRZ data cross-referenced against the Visual Inspection Zone (VIZ) — discrepancies indicate document fraud.

VerfiX validates MRZ check digits on every document where an MRZ is present, cross-references MRZ against VIZ, and retains both the extracted data and validation result in the audit trail. MRZ validation is a core component of FINMA Circular 2016/7 compliance and is performed on every submission — not just suspected fraud cases.

ePassports and modern identity cards contain an NFC chip with biometric data digitally signed by the issuing government. Reading and validating this chip provides the highest assurance level of identity verification — cryptographic authenticity that cannot be physically counterfeited. VerfiX supports NFC chip reading for eIDAS High assurance use cases.

NFC chip reading reads and validates the biometric chip embedded in ePassports (ICAO 9303 compliant) and an increasing number of national identity cards (ISO 18013 compliant). The chip stores: the holder's photograph at high resolution, fingerprints (most passports), personal data from the biographical page, and the issuing government's digital signature over all this data.

Cryptographic validation: The chip data is signed by the issuing government's Certificate Authority, using keys registered in the ICAO Public Key Directory. VerfiX validates this signature chain, confirming the chip data was genuinely issued by the claimed government and has not been modified. A physical counterfeit document — however convincing visually — cannot replicate a valid government digital signature.

When to use NFC: High-value account opening, eIDAS High assurance use cases, anti-spoofing for high-risk customer segments, any scenario where maximum identity assurance is required.

NFC reading requires NFC-capable device and appropriate mobile SDK or web integration. VerfiX supports NFC as an optional enhancement to the standard verification flow.

VerfiX uses computer vision models trained on millions of real and fraudulent documents to detect tampering, forgery, and reproductions. It checks MRZ check digit integrity, security feature consistency, document geometry, ink patterns, hologram placement, and UV-reactive element signatures. The fraud score is included in the Trust Score calculation.

Document fraud detection in VerfiX operates at multiple levels simultaneously:

Mathematical checks: MRZ check digit validation catches any alteration to core data fields. A changed date of birth, document number, or expiry date invalidates the corresponding check digit — automatically flagged.

Security feature analysis: Each document type has defined security features at specific locations — holograms, microprint, kinegrams, UV-reactive inks, security threads. Computer vision models trained on the specific template for each document type verify these features are present, correctly positioned, and consistent.

Geometric analysis: Identity documents have precise dimensional specifications. Scanned or photographed documents submitted as originals show dimensional distortion detectable by the platform.

Reproduction detection: Photographs of photographs create moiré patterns and resolution artefacts detectable in the frequency domain. Printed documents have ink dot patterns characteristic of printing rather than original document production.

Anomaly detection: Statistical models flag documents where data field characteristics — font metrics, spacing, ink density — deviate from the expected distribution for the document type.

All fraud signals are included in the Trust Score and retained in the audit trail.

Document image quality directly affects verification accuracy. VerfiX defines minimum and recommended quality standards:

Minimum requirements: JPEG, PNG, or WebP format; 640×480px minimum resolution; even lighting without glare or shadows; document flat and fully in frame; less than 15 degrees rotation from face-on; text legible to human inspection; file size under 10MB.

Recommended: 1080×1440px or higher resolution; neutral indoor or daylight lighting; document as parallel to camera as possible; MRZ and all data fields fully visible without obstruction; no reflective glare on hologram areas.

Integration guidance: VerfiX recommends providing real-time camera guidance in the integration UI — showing the customer where to position the document, alerting when lighting is insufficient, and confirming capture before submission. This reduces retry rates and improves completion rates.

The API response includes an image quality score alongside the verification result, enabling integrations to implement quality-gating before the final submission. The Integration Guide contains UI guidance patterns for optimised document capture flows.

Yes. VerfiX handles documents in all major scripts including Latin, Cyrillic, Arabic, Chinese, Japanese, Korean, Thai, and others. The MRZ is always in Latin script regardless of document language, providing a reliable data anchor. OCR engines handle script-specific character sets for Visual Inspection Zone data extraction.

Identity documents are issued in the national language(s) of the issuing country. VerfiX's OCR engine handles documents across all major writing systems:

Latin script: All European, Latin American, and many African and Asian countries
Cyrillic: Russia, Ukraine, Bulgaria, Serbia, and others
Arabic: UAE, Saudi Arabia, Egypt, Jordan, Morocco, and others
Chinese (Simplified and Traditional): China, Taiwan, Singapore, Hong Kong
Japanese: Japan (including Kanji, Hiragana, Katakana)
Korean (Hangul): South Korea
Thai: Thailand
Devanagari: India (various)
Georgian, Armenian, and other regional scripts

Critical data anchor: Regardless of the document language, the Machine Readable Zone is always in Latin script following ICAO 9303. This means name, date of birth, document number, and nationality are always extractable from the MRZ even when the VIZ is in a non-Latin script. VIZ extraction in the native script is performed in addition where supported.

Date formats, address formats, and name ordering conventions (family name first in many Asian documents) are handled by document-type-specific parsing rules.
